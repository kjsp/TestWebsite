Usecase

유스케이스 작성 방법
1. 반드시 주어(실행자)을 넣는다.
2. 한번에 실행되는 단위로 글목록을 구분한다.
3. 예외는 소문자로 표기한다.
4. 반드시 사용자 입장에서 작성한다.

대문 페이지 보이기

1. 사용자는 주소를 입력한다.
2. 시스템은 index.jsp를 브라우저에 출력한다.
	a. 시스템 에러시 경고페이지를 출력한다.

로그인
1. 사용자는 ID와 password를 입력한다.
2. 시스템은 ajax를 이용해 id와 password를 DB에서 비교한다.
  2.1 시스템은 id와 password가 일치하는 정보가 DB에 존재시 환영 메제시를 반환한다.
  2.2 시스템은 정보가 일치하지 않을시 경고 메세지를 반환한다.
3. 사용자는 매세지를 확인한다.
  3.1 시스템은 로그인 성공시 main.jsp로 이동한다.
  3.2 사용자는 로그인 실패시 재입력한다.

회원가입
1. 사용자는 index.jsp에서 회원가입 버튼을 누른다.
2. 시스템은 회원 가입 페이지로 이동한다.
3. 사용자는 ID, password, nickname, e-mail을 입력하고 회원가입을 누른다.
  3.1 시스템은 javascript를 이용하여 필수 조건을 체크한다.
  3.2 시스템은 ajax를 통하여 ID의 중복을 체크한다.
4. 시스템은 DB에 정보를 저장하고 메세지를 출력한다.

게시판 글 목록 보기
1. 사용자는 게시판 보기를 누른다.
2. 시스템은 DB에 접근해 글 목록을 가져와 출력한다.(board.jsp)
  2.a 글이 존재하지 않을시 글 등록 부탁 메세지를 출력한다.
3. 사용자는 페이징을 통해 글 목록을 탐색할 수 있다.
4. 시스템은 ajax로 2부터를 반복한다.
